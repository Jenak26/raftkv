What am I looking at?

Each box is a server in a 5-node cluster. The cluster elects exactly one Leader; the rest are Followers. Every write goes through the leader, which replicates it to the others. The magic of Raft: the cluster keeps one consistent log even when servers crash or get cut off - as long as a majority (here, 3 of 5) can still talk.

This is the real consensus code from the repo, not a fake animation. Faults are injected through the same simulated network the test suite uses.

Try this

  1. Hit Propose entry a few times - watch commit and log climb in lock-step on every node (replication).
  2. Crash the green Leader. Within a moment a new leader is elected at a higher term (failover).
  3. Isolate a node, propose more, then Rejoin it - it catches up to the others.
  4. Restart a crashed node or hit Heal all to reset.
